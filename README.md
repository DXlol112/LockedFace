1. Общая информация
    1.1. Название проекта: Удерживания внимания при ра
    1.2. Тип проекта: учебный
    1.3. Уровень: студент 1 курса
    1.4. Язык программирования: Python
    1.5. Тип приложения: desktop-приложение
    1.6. Используемые технологии: Python, библиотека OpenCV
    1.7. Назначение документа: формализация требований к разработке программного приложения

2. Описание задач
    2.1. Необходимо разработать desktop-приложение, определяющее состояние внимания пользователя с использованием веб-камеры.
    2.2. Приложение анализирует изображение лица пользователя в видеопотоке.
    2.3. На основе положения лица и направления взгляда определяется, направлен ли взгляд пользователя в сторону экрана.
    2.4. Результатом работы является определение одного из двух состояний: пользователь внимателен или отвлечён.
    2.5 Если пользователь отвлечён, попытаться вернуть его внимание при помощи сигналов.

4. Функциональные требования
    3.1. Приложение должно получать видеопоток с веб-камеры в реальном времени.
    3.2. Приложение должно выполнять обнаружение лица пользователя на каждом кадре.
    3.3. Приложение должно определять направление взгляда пользователя относительно экрана.
    3.4. На основе анализа должно устанавливаться одно из состояний:
       – «внимателен» — лицо обнаружено и направлено в сторону камеры; 
       – «отвлечён» — лицо не обнаружено либо отклонено от камеры.
    3.5. Если состояние <<отвелчён>> вернуть его внимание при помошью сигналов взоимодействия с пользователем.
    3.6. Приложение должно подсчитывать суммарное время нахождения пользователя в состоянии «отвлечён».
    3.7. Пользователь должен иметь возможность завершить работу приложения стандартным способом (кнопка закрытия окна или клавиша).
    3.8 Пользователь может выбирать громкость, время работы програмы, менять сигналы на выбраные.

6. Нефункциональные требования
4.1. Приложение должно работать под управлением операционной системы Windows.
4.2. Обработка видеопотока должна выполняться с частотой не ниже 15 кадров в секунду при наличии лица в кадре.
4.3. Приложение должно завершать работу корректно без аварийного завершения и зависаний.
  4.4. Исходный код должен быть структурирован и разбит на логические части.
  4.5. Допускается использование только открытых и свободно распространяемых библиотек Python.

5. Требования к интерфейсу
    5.1. Интерфейс приложения должен состоять из одного основного окна.
    5.2. В основном окне должно отображаться изображение с веб-камеры и настройками.
    5.3. Интерфейс не должен содержать элементов, не влияющих на работу приложения.
    5.4. Управление приложением должно быть интуитивным и не требовать обучения пользователя.

6. Входные и выходные данные
    6.1. Входные данные:
       – видеопоток с веб-камеры пользователя в реальном времени.
    6.2. Выходные данные:
       – отображаемое видео с камеры;
       – если пользователь отвлёкся попытатся вернуть;


7. Критерии приемки
  7.1. Приложение успешно запускается на компьютере под управлением Windows.
  7.2. Видеопоток с веб-камеры отображается в окне приложения.
  7.3. Лицо пользователя обнаруживается при нахождении в зоне видимости камеры.
  7.4. Состояние «внимателен» и «отвлечён» определяется и отображается корректно.
  7.5. Время невнимательности корректно увеличивается при соответствующем состоянии.
  7.6. Приложение стабильно работает не менее 5 минут без сбоев.

8. Дополнительные условия
  8.1. Проект выполняется одним студентом.
  8.2. Допускается использование учебных примеров и официальной документации OpenCV.
  8.3. Итоговый результат должен включать исходный код и текстовую инструкцию по запуску приложения.
  8.4. Проект разрабатывается исключительно в учебных целях.
